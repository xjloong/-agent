# 多模态客服答案提分规律与制作注意事项

本文档根据多轮提交分数变化反推整理，用于后续人工或半自动制作答案时参考。它不是官方评分细则原文，而是结合实际提分结果总结出的执行优先级。

## 一、最高优先级：图文必须精准匹配

这是目前最明确、最有效的提分方向。

1. 图片必须直接服务于旁边的文字  
   题目问部件位置，就配部件编号图；题目问步骤，就配步骤图；题目问状态灯、表格、规格，就配对应表格或界面图。

2. `<PIC>` 不要统一堆在答案最后  
   图片应插在它解释的那句话或那一步之后。  
   例如：  
   - “按下释放按钮取出电池组，释放按钮位置见图。<PIC>”
   - “红灯闪烁表示正在充电，指示灯状态见图。<PIC>”

3. 多图可以加分，但前提是每张图都有明确用途  
   多图不是问题，错图才是问题。  
   如果一个流程有 4 个关键步骤，且手册有 4 张对应图，就应该分步骤插入 4 张图。

4. 不要配泛泛产品图  
   问“卸载驱动程序”，不应放鼠标外观图。  
   问“损害免责”，不应只放保修年限表，除非答案明确说明该表只是保修政策背景。

## 二、第二优先级：答案必须严格命中问题范围

LLM 裁判很看重“是否直接回答问题”。

1. 问“前五条”，只答前五条  
   不要扩展成八条，也不要混入其他章节的内容。

2. 问某个具体章节，就不要答相邻章节  
   例如“连接电源时注意事项”应来自“连接电源时”章节，而不是混入安装、清洁、日常使用注意事项。

3. 问具体型号、具体模式、具体故障时，要保留限定条件  
   型号、模式、按钮名、指示灯颜色、单位、时间、温度、距离等，尽量按手册准确写出。

4. 如果手册有多个方案，要说明适用场景  
   例如 BIOS 更新既有 EZ Flash，也有 DOS BIOS Updater，题目问 “update the BIOS file” 时，应覆盖官方给出的主要方法，而不是只答其中一个。

## 三、第三优先级：步骤题要写成完整流程

“如何 / how to / steps / procedure / install / replace / clean / set / connect” 这类题最容易因为答案太概略而拿不到高分。

制作要求：

1. 按实际操作顺序写  
   先准备，再操作，再确认，再补充安全提醒。

2. 关键动作后插图  
   例如拆盖、拔销、插入电池、选择菜单、确认灯光状态，都应尽量有图对应。

3. 不要只写“按照手册步骤操作”  
   要把步骤展开。  
   “按滤网安装步骤更换滤网”这种表达不够好，应写清楚取上盖、取滤网盖、拆包装、装滤网、装回盖板、重置指示灯。

4. 最后补一句确认条件  
   例如“确认螺丝拧紧”“确认指示灯熄灭”“确认部件卡入到位”“确认设备可正常运行”。

## 四、第四优先级：表格、状态灯、规格类问题要充分利用表格图

这类题天然适合图文并茂。

优先配图对象：

1. LED 指示灯状态表
2. 故障排除表
3. 规格参数表
4. 保修期限表
5. 心率区间表
6. BIOS/菜单界面截图
7. 遥控器、控制面板、按钮编号图

写法建议：

先给结论，再说明表格含义，并把 `<PIC>` 放在表格说明旁边。  
例如：“下图表格对应基座 LED 行为和当前状态，可按 steady on、flashing、fast flashing 三类判断。<PIC>”

## 五、第五优先级：纠正手册 OCR 或排版问题

老师提示说明书可能存在错别字或排版问题，所以不能机械复制。

处理原则：

1. 保留手册事实，不保留明显乱码
2. 修正明显错别字、断句、OCR 误识别
3. 单位、数字、型号要谨慎，不确定时回查原图或上下文
4. 英文手册 OCR 错误可以整理成通顺英文

示例：

“Afl east 4 fee fin front” 应整理为 “at least 4 feet in front”。

## 六、第六优先级：答案结构要清晰，但不要只做表面格式

结构清晰有帮助，但单纯格式包装提分有限。

推荐结构：

1. 首句直接回答问题
2. 中间按步骤或分类展开
3. 每个关键步骤配对应 `<PIC>`
4. 结尾补安全提醒、适用条件或确认方式

不推荐：

1. 只加“首先、其次、最后”，但内容没有补全
2. 只加“见图”，但图片和文字没有强对应
3. 为了变长加入无关背景

## 七、第七优先级：适度丰富答案，不要无效冗长

从测试结果看，单纯加长不一定提分，甚至可能降分。

有效扩写包括：

1. 补充手册中的限制条件
2. 补充安全警告
3. 补充故障判断条件
4. 补充操作后的确认动作
5. 补充不同型号或不同模式的区别

无效扩写包括：

1. 泛泛客服安慰话
2. 与问题无关的产品介绍
3. 重复同一句话
4. 把相邻章节全部复制进去

## 八、第八优先级：语言必须匹配问题

中文题用中文回答，英文题用英文回答。

不要出现：

1. 中文问题英文答
2. 英文问题中文答
3. 中英文混杂导致客服体验差

如果手册是英文但问题是中文，应翻译成自然中文；如果问题是英文，应用自然英文回答。

## 九、第九优先级：客服类问题边际收益较低

从分开测试结果看，前 50 条客服套话不是主要提分来源。

客服题只需保证：

1. 直接回应用户诉求
2. 语气礼貌
3. 处理方案明确
4. 必要时索要订单号、地址、维修单号等信息
5. 不承诺手册或规则中没有的权益

主要提分空间仍在产品手册题。

## 十、提交文件格式是硬性底线

格式不对会直接影响提交，不属于可选优化。

每次提交前必须检查：

1. 总行数 400 条答案，加表头共 401 行
2. id 顺序与题目文件一致
3. 客服题为纯文本
4. 产品题格式为 `"文本", ["图片id"]`
5. `<PIC>` 数量必须等于图片列表数量
6. 图片 id 必须真实存在
7. 不允许出现连续 `<PIC><PIC>`
8. 不允许有空答案
9. 不允许有乱码替换符 `�`
10. 不需要压缩包，直接提交 CSV

## 十一、推荐的后续制作流程

如果继续冲高分，建议按以下顺序排查：

1. 产品题无图答案  
   优先查是否有强相关图；没有强相关图才保留无图。

2. 步骤题但答案很短  
   检查是否漏了完整流程或关键步骤图。

3. 长答案但只有一张图  
   检查手册同章节是否有多个对应图。

4. 图片只在结尾的答案  
   尽量把图移动到对应步骤或解释旁边。

5. 答案里出现 “see Figure / 参见 / 如图” 但图片没有配全  
   这通常说明漏图。

6. 问题带限定词的答案  
   如“前五条”“首次使用”“某型号”“某模式”“某故障”，逐字核对范围。

7. 表格类问题  
   优先确认是否使用了表格图，而不是只复述文字。

8. 低分风险行人工复核  
   逐题看问题、答案、手册原文、图片，确认是否真的做到图文互补。

## 十二、核心结论

目前最可靠的提分公式是：

准确命中问题范围 + 手册证据完整 + 步骤充分展开 + 图片插在对应解释旁边。

不要把主要精力放在泛泛润色、客服套话、机械加长上。真正能把 3 分或 4 分答案推向 5 分的，是把每道产品题做成“客服能直接照着指导用户、图片能立刻帮助理解”的答案。
